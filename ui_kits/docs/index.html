Ingesting metrics

Send your first spans to Futurist and see them in the console within 800ms. This guide covers the SDK, the wire format, and back-pressure.

Updated Jul 2026·6 min read·v2.4
You'll need an API key with ingest:write scope. Generate one under Settings → API keys.

Install the SDK

The client is distributed as a single package with no runtime dependencies. Install it with your package manager of choice:

terminal copy 
$ npm install @futurist/telemetry

Configure the client

Initialize once at process start. The client batches spans and flushes on an interval or when the buffer fills — whichever comes first.

client.ts 
import { Telemetry } from '@futurist/telemetry';

const t = new Telemetry({
  key: process.env.FUTURIST_KEY,
  region: 'us-east-1',
  flushMs: 800,        // max buffer age
  maxBatch: 4096,      // spans per flush
});

Send a span

Every span needs a series, a timestamp, and a numeric value. Labels are optional but power the console's grouping and filters.

  • series — the metric name, e.g. http.request.duration
  • value — a float; the unit is inferred from the series suffix
  • labels — low-cardinality key/values only (avoid user IDs)

Cardinality matters

Labels multiply. A single high-cardinality label — a request ID, a full URL — can explode a series into millions of variants and dominate your bill. Keep labels bounded: regions, status codes, service names. If you need per-request detail, use a trace, not a label.

Handling back-pressure

When the collector is saturated it returns 429 with a Retry-After header. The SDK honors it automatically with exponential backoff, so you rarely need to handle this yourself — but if you run your own transport, respect the header.
